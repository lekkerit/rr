# **Section 1 — Course Architecture**

## **The Core Problem You Solve**

Most people using Claude.ai today are stuck in stateless mode. Every session starts from scratch. They type the same context repeatedly. They get good outputs but can't build on them. They feel like they're using a powerful tool badly.

Bots in Public moves people from casual prompters to builders — from stateless to stateful, from one-off outputs to a real project they own, understand, and can grow.

*This is the narrative spine of the entire course. The tool progression IS the story.*

## **Day 3 — The Stepping Stone**

| 🧊 Day 3: The folder remembers. Claude still doesn't. Here's why that matters. |
| :---- |

You've been using Claude in the browser. The memory lives in your CLAUDE.md file — not in Claude. You paste it in every time. Claude borrows it.

There's a tool called Claude Cowork. It lets you point Claude at a local folder on your computer — so it can read your files directly. No pasting. This feels like a big upgrade. But here's the honest truth:

| Cowork is stateless too. The folder persists. Claude doesn't. Every session Claude still starts fresh — it just reads your files faster. The folder remembers. Claude borrows it. This is the stepping stone, not the destination. |
| :---- |

The destination is Claude Code — where Claude actually runs inside your project, reads your CLAUDE.md automatically, and carries context across everything. That's Week 2\. That's the cure.

Tomorrow: we set up VS Code — the editor you'll use for everything in Week 2\.

## **Day 4 — VS Code \+ Project Structure**

| 💻 Day 4: Set up the environment. Week 2 needs it ready. |
| :---- |

Today is practical. No new concepts. Just setting up the workspace Week 2 depends on.

**Step 1 — Confirm VS Code is open**

If you installed it on Day 1, open it. If not: code.visualstudio.com — download, install, open.

**Step 2 — Open your bots-in-public folder**

File → Open Folder → find bots-in-public on your Desktop. You should see your CLAUDE.md in the left panel.

**Step 3 — Create your project structure**

Inside bots-in-public, create two folders: notes and output. Your folder should now look like this:

| bots-in-public/└── CLAUDE.md└── notes/└── output/ |
