## Section 1: TikTok / YouTube Shorts Script

**[HOOK — 0-3s]**
"I built an AI system that runs a business this week. And I didn't write a single line of code myself."

**[PROBLEM — 3-10s]**
Most restaurants respond to Google reviews wrong. Copy-paste replies. Generic "thank you for your feedback." Customers can smell it immediately — and it kills trust.

**[SOLUTION — 10-25s]**
I'm building Review Recovery. The AI reads every Google review, understands the tone, and writes a response that sounds like the actual owner wrote it. Warm. Specific. Human.

But here's the thing — I'm not a developer. So this week I built something that handles the complexity for me.

**[THE FOLDER STRUCTURE — 25-40s]**
I set up a Claude Code project structure. Think of it like giving your AI a proper office:
- A memory so it remembers everything about the project — no re-explaining every session
- Scripts that handle the repetitive stuff reliably, every single time
- A folder of instructions — like an employee handbook — so the AI follows the same process, not a different one each time

Now when I open a session, Claude already knows the product, the customers, the tone, the tools. It just... picks up where we left off.

**[PROOF — 40-55s]**
What shipped this week:
- 3 professional Instagram reels — scripted, animated, exported as video. Fully AI-assisted.
- A complete database design: 9 tables tracking restaurants, reviews, AI responses, approvals, analytics
- The full automation chain: Google reviews come in, AI writes the reply, I approve it, it posts

**[CTA — 55-60s]**
I'm building this in public. Follow along if you want to see how far AI can take a one-person business — no coding required.

## Section 2: YouTube Long-Form Script Outline

**Title ideas:**
- "I used AI to build an entire business system this week (no code)"
- "How I gave my AI a memory — and why it changed everything"
- "Building a real AI product in public: week 1 recap"

**Intro (0-2 min)**
Set the scene: Review Recovery is an AI tool for restaurants. The goal is to respond to Google reviews automatically, in a way that sounds human. But building it solo means you need AI to help build the builder.

**The problem with AI tools (2-4 min)**
Every session, you start from scratch. You have to re-explain the context, the product, the rules. It's like hiring someone with no memory. This week I fixed that.

**The rr/ folder structure — what it actually means (4-8 min)**
Walk through the project structure visually. Explain in plain English:

1. **Memory** — `CLAUDE.md` is a file Claude reads at the start of every session. It contains everything about the product: who it's for, how it works, what the rules are. Claude never forgets.

2. **Scripts** — Instead of AI improvising (which leads to errors), I have scripts that handle the mechanical work: fetching reviews, generating Instagram slides, exporting video. AI calls these scripts. They're reliable.

3. **Directives** — The `directives/` folder is like an employee handbook. Each file is a standard process: "here's how to create an Instagram post, step by step." AI reads it and follows it — consistently.

4. **The result** — AI can now handle full workflows end to end. I describe the goal, it reads the instructions, runs the right tools, checks the output, and improves the process if something breaks.

**What shipped this week (8-12 min)**
Go through each deliverable:
- 3 Instagram reels (show them)
- Airtable database — walk through what 9 tables actually means for a business
- The automation chain — show how a Google review becomes an approved response

**Why this matters for non-developers (12-15 min)**
You don't need to code this. The system does the heavy lifting. You need to:
1. Know what you want to build
2. Describe it clearly
3. Review the output

AI does the boring part. You stay in control.

**Outro + CTA**
Building in public. Drop a comment if you want to see a specific part of the build. Subscribe for weekly updates.
